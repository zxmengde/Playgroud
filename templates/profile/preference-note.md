# 偏好记录

# source

偏好来源，例如用户明确反馈、任务产物修改、模板文件、项目规范或重复出现的选择。

# domain

适用领域：科研、Word、PPT、Excel、编码、网页、知识库、协作方式或其他。

# preference

可复用偏好内容。只记录稳定、有来源、会影响未来任务的偏好。

# confidence

采用以下状态之一：

- confirmed：用户明确确认。
- inferred：从多次任务中低风险推断。
- pending：需要后续确认。

# scope

适用范围和不适用范围。

# update_targets

需要同步更新的文件、模板、技能或工作流。
