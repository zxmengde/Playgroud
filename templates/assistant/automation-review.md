# 自动化评估记录

# name

自动化名称。

# purpose

要减少的重复工作或要提高的检查可靠性。

# trigger

运行频率、触发方式或适用场景。

# inputs

需要读取的本地文件、规则、技能、网页或外部状态。

# outputs

输出到当前线程、Triage、本地任务记录、报告文件或其他位置。

# permissions

允许操作、需任务级授权或预授权的操作，以及禁止操作。

# validation

如何判断自动化是否正确完成；失败时如何被发现。

# risk_review

可能的误报、漏报、噪声、隐私、账号写入、文件覆盖和长期运行风险。

# decision

采用以下结论之一：

- 不创建，仅保留人工流程。
- 先改为 skill、模板或脚本。
- 创建线程心跳。
- 创建独立定时任务。
- 暂停或删除已有自动化。

# review_after

何时复盘该自动化是否仍有价值。
