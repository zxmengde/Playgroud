# Skill 评估与迁移记录

# skill

待评估 skill 名称或来源。

# source

来源 URL、仓库、文档或用户授权材料。

# task_fit

该 skill 试图解决的真实任务，以及是否在当前用户工作中重复出现。

# useful_parts

可迁移的原则、模板、脚本思路、验证方式或反例。

# risks

权限、隐私、外部网络、脚本执行、账号写入、版本过期、风格不匹配和误触发风险。

# migration_decision

采用以下结论之一：

- 不迁移，仅记录为资料。
- 迁移为本地规则。
- 迁移为模板。
- 迁移为脚本。
- 迁移为自有 skill。
- 待用户授权或补充资料。

# validation

格式校验、脚本检查、代表性任务、截图、测试或人工审查结果。

# follow_up

后续需要采集的用户偏好、样例文件、模板路径或权限确认事项。
