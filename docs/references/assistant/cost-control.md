# 成本控制与输出纪律

本文件用于减少无效 token 和费用，同时保持判断、验证和产物质量。成本控制的目标是去掉冗余，不是降低能力。

## 基本原则

默认采用紧凑表达：少铺垫、少重复、少泛泛解释。简单任务用短答；复杂任务只保留目标、关键判断、产物路径、验证结果和剩余风险。

不得为了省 token 跳过必要步骤。涉及科研引用、代码修改、外部资料、权限、安全、金融、法律、医学或正式文档时，仍应完成来源核验、测试、渲染、截图或人工可读检查。

用户提到第三方“少说话”类 skill 时，默认只吸收可迁移原则：简洁、直达、避免复述、用文件产物承载细节。不得安装来源不明或会降低判断质量的第三方 skill。

## 执行规则

读取上下文时采用渐进方式。先读取入口规则、任务相关文件和索引；只有当任务需要时，再读取长文档、历史记录或外部资料。避免把可由文件路径说明的内容大量复述到最终回答中。

工具调用应以完成任务为目标。可并行的只读检查可以并行；不要反复运行不会改变结论的命令。外部检索只在需要当前事实、来源证据或用户明确要求时执行。

输出应优先给出可用产物。能写入文档、脚本、知识条目或补丁的内容，不应用长篇对话替代。最终回复通常只需要说明改了什么、验证了什么、还有什么风险。

进展更新应简短。长任务每个自然检查点说明正在做什么、发现了什么、下一步执行什么；不展开完整推理。

## 不能省略的内容

以下内容即使增加 token，也不能省略：

- 会改变结论的关键假设和证据边界。
- 文件修改路径和验证命令。
- 引用、论文、网页和视频资料的来源。
- 失败命令、阻塞原因和解除条件。
- 涉及删除、覆盖、账号、发布、购买、提交表单或敏感信息时的确认要求。

## 停止前检查

结束前检查是否存在以下问题：回答比任务需要更长；重复解释用户已知道的内部规则；把计划当产物；把可以写入文件的内容只放在对话中；为了简短而省略验证或来源。
