# 偏好采集问卷

本问卷用于在初始化文档、技能或复杂任务前采集用户偏好。Codex 应先读取已有用户画像，再只询问缺失且会影响当前任务质量的问题。

## 使用原则

不要一次性把全部问题机械抛给用户。根据任务类型选择相关部分。若用户要求完整采集，则按模块分批访谈，每批最多三个高影响问题。

采集结果应写入 `docs/profile/user-model.md`，必要时同步更新相关工作流、技能或模板。

## Office 偏好

Word：

- 是否存在常用 `.docx` 模板；若有，路径是什么。
- 默认字体、字号、标题层级、页边距和编号规则。
- 引用格式和参考文献格式。
- 图表标题、表格样式和图片说明格式。
- 是否需要目录、摘要、附录和修订记录。

PPT：

- 是否存在常用 `.pptx` 模板；若有，路径是什么。
- 默认比例、字体、配色和页边距。
- 每页信息密度偏好。
- 图表风格、流程图风格和图片使用偏好。
- 适用场景：组会、开题、中期、答辩、企业汇报或课程展示。

Excel：

- 默认表格样式、颜色、冻结窗格和筛选规则。
- 图表类型偏好。
- 是否需要公式说明、数据验证、透视表或仪表盘。

## 科研偏好

- 主要研究方向和长期课题。
- 常用数据库、期刊、会议、标准和专利来源。
- 常用引用格式和文献管理工具。
- 输出默认长度和结构。
- 对证据强度、方法局限和不确定性的呈现要求。
- 导师、组会、论文、企业材料等不同受众的差异。

## 编码偏好

- 常用语言、框架和项目目录。
- 测试、格式化、静态检查和提交习惯。
- 代码修改偏好：小范围修复、结构优化、重构边界。
- 是否允许自动创建分支、提交和推送。

## 网页与资料偏好

- 默认是否截图。
- 下载文件默认保存位置。
- 网页资料如何命名和归档。
- 是否需要同时生成 Markdown 摘要、PDF、截图或知识条目。

## 知识库偏好

- 是否与 Obsidian、Zotero、MEMS 项目或其他工具联动。
- 知识条目命名规则。
- 标签体系。
- 哪些内容需要长期保存，哪些只保留在任务记录中。
