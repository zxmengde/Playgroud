# 偏好地图

本文件记录 Codex 对用户偏好的已知、默认和待采集状态。它用于防止 Codex 在缺少模板、审美、科研标准或工作习惯时假装已经了解用户。

## 使用规则

复杂任务开始前，Codex 应先检查本文件和 `docs/profile/user-model.md`。若待采集偏好会显著影响产物质量，应使用 `preference-intake` 进行少量高影响访谈。若用户不想回答，应采用保守默认值，并把未知项写入任务记录或本文件。

偏好采集不是让用户填写表格。问题应帮助理解用户如何判断质量、如何做取舍、对什么结果敏感、希望 Codex 何时主动扩展、何时必须停下确认。

## 访谈层级

设计问题时，优先从以下层级选择：

- 目标层：这项工作最终要改善什么结果。
- 判断层：用户如何判断产物有用、可信、专业或不合格。
- 证据层：什么来源、实验、引用或检查可以支撑结论。
- 表达层：面向谁，应该采用什么语气、结构、密度和视觉形式。
- 流程层：用户希望 Codex 直接做哪些部分，哪些部分先确认。
- 风险层：哪些错误会造成较大返工、误导或外部影响。

字段型问题只在确实需要时询问，例如模板路径、页数、比例、引用格式、测试命令和输出目录。

## 已知默认值

以下默认值已由用户反馈和现有任务确认：

- 语言：简体中文。
- 语气：客观、严谨、平实、克制、专业、冷静、中立。
- 工作方式：质量优先，复杂任务先理解真实需求，再持续推进到产物、验证或明确阻塞。
- 系统负担：用户不希望记忆内部文件名、技能名或维护流程，Codex 应主动承担同步、读取、验证、记录和改进建议。
- 主动性：用户希望 Codex 举一反三，在低风险范围内主动扩展有价值的工作。
- 判断姿态：用户不希望 Codex 迎合或机械执行。应检查用户观点、需求和前提是否成立，并识别可能的伪需求。
- 成本控制：用户希望减少无效 token 和费用，但不接受以降低判断质量、验证深度或产物质量为代价。
- 夜间或离开时：Codex 可继续低风险、可审计、可恢复的调研、整理、校验、提交和同步。

## Office 偏好状态

已知：

- 默认采用学术科研风格，强调结构清晰、证据充分、图表规范、可编辑性和可验证性。
- 适用场景包括论文、组会、导师沟通、科研汇报和项目材料。

待采集：

- Word 常用模板、字体、字号、标题层级、页边距、引用格式、图表编号和目录规则。
- PPT 常用模板、比例、版式、配色、字体、信息密度、图表风格、图片使用偏好和受众类型。
- Excel 表格样式、图表偏好、公式可读性、数据验证、透视表和说明页要求。

保守默认：

- Word 使用清晰标题层级、普通学术段落、可编辑文本和可核验引用。
- PPT 使用高对比、低装饰、清晰层级、可编辑图表和适合科研汇报的信息密度。
- Excel 使用可读表头、筛选、必要公式说明和数据来源说明。

## 科研偏好状态

已知：

- 科研输出默认同时服务证据型笔记和可发表文本。
- 需要区分事实、推断和不确定性。
- 重要结论应优先使用论文、官方文档、标准、数据集和机构资料。
- 用户希望支持 Zotero、本地 PDF、文献检索、论文写作和引用核验。

待采集：

- 长期研究方向的更细分问题、导师偏好、目标期刊或会议、引用格式、文献管理工具、常用数据库、输出长度和图表偏好。
- Zotero 数据目录、常用集合、标签规则、是否安装 Better BibTeX、是否允许只读读取本地库、是否允许使用 Zotero Web API。

保守默认：

- 以研究问题、证据链、方法局限、后续任务组织内容。
- 引用优先保留 DOI、出版社页面、arXiv、PubMed、IEEE、ACM、标准机构或数据集页面。

## 编码偏好状态

已知：

- 先理解项目结构，再修改。
- 遵循项目已有风格。
- 不回退用户已有改动。
- 运行相关测试或说明未能测试原因。

待采集：

- 常用语言、项目目录、提交粒度、提交信息风格、分支习惯、测试优先级、重构边界和自动推送偏好。

保守默认：

- 小范围修改，避免无关重构。
- 优先运行最相关的测试、构建或静态检查。
- 提交前检查 `git status` 和差异。

## 网页与知识偏好状态

已知：

- 需要当前资料时应联网核对。
- 长期有价值的资料应沉淀为本地知识条目。
- 登录、提交、下载敏感资料或外部写入前确认。
- 用户希望支持 Bilibili 等视频资料总结，优先使用字幕、元数据和可核验页面信息。

待采集：

- 默认是否截图、下载目录、网页摘录命名规则、是否同步到 Obsidian/Zotero/MEMS、标签体系和归档深度。
- 视频摘要默认是否需要时间戳、是否保存字幕文件、是否允许下载音频或视频、是否允许登录后读取字幕。

保守默认：

- 保留 URL、访问日期、关键摘录、来源可信度、不确定性和本地知识条目路径。

## 需要后续从真实任务中学习的信号

Codex 应在真实任务中观察以下信号，并在稳定后请求用户确认是否写入长期偏好：

- 用户反复修改的文字风格、图表风格和结构。
- 用户认可或否定的 PPT 页面密度、配色和叙事方式。
- 用户对引用、证据、推断和结论强度的修改。
- 用户对代码改动范围、测试深度和提交方式的反馈。
- 用户对问题数量、问题深度和主动扩展程度的反馈。
