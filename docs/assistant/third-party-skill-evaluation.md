# 第三方 Skills 评估准则

本文件用于指导 Codex 评估外部 skills、插件、agent 模板和市场目录。默认策略是先研究和抽取可迁移方法，不直接安装。

## 基本结论

第三方 skills 的价值取决于任务匹配度、上下文兼容性、版本时效、权限边界和验证方式。公开技能数量增长很快，但数量本身不能代表质量。若技能与当前用户、项目、工具链或任务验收方式不匹配，可能增加上下文负担，甚至降低结果质量。

## 可参考来源

优先参考以下来源：

- 官方技能仓库和官方文档。
- 有清晰许可证、维护记录和示例任务的开源仓库。
- 有测试、脚本和明确输入输出的技能。
- 有安全说明、权限边界和外部访问说明的技能。
- 有独立评估或 benchmark 结果的技能。

谨慎处理以下来源：

- 匿名市场目录。
- 需要下载压缩包或运行安装脚本但缺少源码说明的技能。
- 声称解除工具限制、绕过权限或访问外部账号的技能。
- 涉及个人聊天记录、同事资料、邮件、网盘或组织知识库采集的技能。
- 围绕泄露源码、破解版本或未授权镜像传播的技能。

## 审查步骤

评估一个第三方 skill 前，Codex 应先完成以下检查：

- 读取 `SKILL.md`，确认 name、description、触发场景和禁止行为。
- 检查脚本目录，识别文件读写、网络访问、外部命令、删除或上传行为。
- 检查是否要求密钥、令牌、账号密码或长期授权。
- 检查技能内容是否与仓库主题一致，避免描述无害但仓库行为不一致。
- 检查是否有测试文件、示例输入输出、版本记录和失败说明。
- 检查是否会把不可信网页、邮件或工具返回内容当作指令。

## 迁移方式

默认迁移方式不是安装，而是抽取原则：

- 若技能提供了有用流程，把流程改写为本仓库的 workflow 或 skill。
- 若技能提供了脚本思路，先在本地用小样本重写和验证。
- 若技能提供了模板，先转换为本仓库 `templates/` 下的可审计模板。
- 若技能依赖外部账号，先写入权限规则，等待用户明确授权。
- 若技能涉及隐私数据采集，只能在用户明确授权、限定范围和记录用途后考虑。

## 验收方式

一个外部 skill 只有在通过以下检查后，才可考虑成为本系统能力的一部分：

- 能明确说明适用任务和不适用任务。
- 能给出输入、输出和验收标准。
- 不保存或复述敏感信息。
- 不绕过权限或账号边界。
- 能通过本仓库校验脚本或专门测试。
- 在至少一个真实或代表性任务中表现出可观察收益。

## 对当前系统的影响

当前系统应优先强化自有 skills，而不是批量安装外部 skills。适合后续吸收的方向包括：科研引用核验、PPT 结构化编辑、Word 样式检查、网页资料安全提取、长期任务状态保存、技能误触发诊断和自我复盘。
