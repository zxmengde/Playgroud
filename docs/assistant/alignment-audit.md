# 对齐审查记录

本文件记录当前控制仓库与用户真实需求之间的差距，以及已采取的修正措施。

## 已发现问题与修正

| 问题 | 影响 | 修正 |
| --- | --- | --- |
| 文档和技能曾主要依据通用流程创建，缺少用户画像 | 容易生成看似合理但不贴合用户的规则和产物 | 新增 `docs/profile/user-model.md` 和偏好采集机制 |
| Office 工作流缺少模板、字体、版式、受众、引用格式等个人偏好 | PPT、Word、Excel 产物可能不符合用户实际审美和科研场景 | Office 工作流改为先读取用户画像，偏好缺失时使用 `preference-intake` |
| 科研工作流只规定通用来源标准，缺少输出形态偏好 | 可能只生成笔记或只生成文本，不能兼顾证据积累和正式写作 | 用户画像中记录“证据型笔记和可发表文本并行” |
| 编码工作流缺少用户测试、重构和提交偏好 | 可能过度修改或测试不足 | 编码工作流要求检查用户画像，缺失关键偏好时先采集 |
| 网页资料流程缺少截图、下载、归档命名偏好 | 资料留存方式不稳定 | 网页工作流要求偏好缺失时先采集 |
| 知识库缺少标签、命名和外部工具联动偏好 | 长期资料可能难以检索和复用 | 新增知识库偏好采集项 |
| 教程仍可能让用户理解内部机制 | 增加用户负担 | 教程改为自然使用说明，内部机制由 Codex 自行承担 |
| 自我改进机制偏被动 | 需要用户主动指出文件或技能名 | 改为 Codex 发现偏差后主动提出系统更新确认 |
| 夜间长任务可能因缺少用户回复而停止 | 用户希望 Codex 在低风险范围内继续工作 | 用户画像增加夜间低风险持续工作规则，并把停止前检查纳入复杂任务 |
| 编码与办公流程对外部工具经验吸收不足 | 可能忽略项目地图、结构化验收和 Windows 自动化边界 | 新增 agent 工具图谱，更新编码流程、办公流程和工具登记 |

## 仍待采集的用户信息

以下信息尚未充分采集。Codex 在相关任务前应主动询问，不得臆测：

- Word 模板、字体、标题层级、引用格式、图表规范。
- PPT 模板、比例、字体、配色、版式、信息密度。
- Excel 表格和图表偏好。
- 科研引用格式、常用数据库、目标期刊或导师偏好。
- 编码测试命令、提交风格、重构边界。
- 网页资料截图、下载、归档命名方式。
- Obsidian、Zotero、MEMS 项目等知识库联动方式。
- 用户允许 Codex 在夜间或长任务中继续执行的任务类型和停止条件。

## 审查结论

当前系统已经从“通用规则优先”改为“用户画像和真实需求优先”。但用户画像仍处于早期状态，后续真实任务中应持续采集偏好，并把采集结果写入 `docs/profile/user-model.md` 和相关模板。
