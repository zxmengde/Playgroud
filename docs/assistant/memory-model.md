# 分层记忆模型

本文件规定 Codex 如何在本地工作系统中保存、读取和修正长期上下文。目标是减少用户重复说明，同时避免把未经确认的推断写成事实。

## 记忆层级

Codex 应区分以下记忆层：

- 当前任务状态：写入 `docs/tasks/active.md`，保存正在进行的目标、已完成内容、下一步和阻塞。
- 用户画像：写入 `docs/profile/user-model.md`，只记录稳定偏好、长期工作场景和已经确认的反向偏好。
- 任务知识：写入 `docs/knowledge/items/`，保存科研、项目、网页资料、办公流程和可复用结论。
- 系统复盘：写入 `docs/assistant/harness-log.md`，记录误解、验证不足、执行不足、工具失败和修正动作。
- 工具与权限：写入 `docs/assistant/tool-registry.md` 和 `docs/assistant/permissions.md`，保存工具用途、风险边界和替代路径。
- 模板与脚本：写入 `templates/` 和 `scripts/`，保存可复用产物和可运行检查。

## 写入规则

当信息满足以下任一条件时，应考虑写入本地记忆：

- 能减少用户未来重复说明。
- 会影响科研、办公、编码、网页或知识管理产物质量。
- 是用户明确表达的偏好、反感行为、质量标准或长期目标。
- 是一个任务中被验证过的来源、结论、路径、命令或模板。
- 是一次失败或低效的原因及修正方式。

不得写入密钥、令牌、账号密码、未授权个人资料或无法说明来源的敏感信息。

## 不确定性处理

用户未明确确认的推断应标记为“待确认”或“低风险假设”。涉及模板、引用格式、审美、导师偏好、账号接入和外部发布时，不得把推断当作稳定事实。

当用户要求暂停提问时，Codex 应停止追问。可以继续基于已有文件、公开资料和任务反馈做低风险改进，并把高影响未知项记录为待采集。

## 读取规则

复杂任务开始前，Codex 应读取用户画像、执行契约、权限规则和相关工作流。若任务涉及长期背景，应检索知识库索引和当前任务记录。

读取记忆时应判断时效性。过期的软件版本、网页信息、政策、价格、模型能力和工具说明需要重新核对。

## 修正规则

用户指出偏差时，应先修正当前任务。若偏差会影响未来任务，应记录到复盘文件，并在普通语言中请求用户确认是否更新系统规则。用户确认后，Codex 自动修改文件、运行校验、提交并同步。

当不同记忆互相冲突时，以更具体、更近期、更接近用户明确表达的信息为准；同时记录冲突并在需要时向用户确认。
