# 连续完成契约

本契约用于防止任务停在计划、建议或中间结果。

## 基本要求

当用户提出明确目标时，应直接推进到以下结果之一：

- 已生成或修改的文件。
- 已完成的代码补丁和检查结果。
- 已完成的资料检索、摘要、来源记录和知识条目。
- 已完成的文档、表格、演示文稿或 PDF 处理结果。
- 已完成的网页访问、资料提取、截图或下载前检查。
- 明确阻塞说明，包括已完成内容、未完成内容、阻塞原因和解除条件。

## 禁止模式

不得在可执行时只给笼统建议。不得用“可以继续”“后续建议”“下一步可做”替代已经可以执行的工作。不得把任务拆解作为停止理由。

## 进展报告

长任务应简短说明正在做什么、发现了什么、接下来执行什么。报告应服务于用户理解任务状态，不替代实际执行。

## 验证要求

凡是能验证的结果，应执行验证。验证方式包括命令输出、测试、构建、渲染、截图、文件结构检查、链接检查、来源核对或人工可读摘要。

无法验证时，应说明原因和剩余风险。
