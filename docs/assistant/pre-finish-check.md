# 停止前检查

本文件用于防止复杂任务在未完成、未验证或未同步时过早结束。它是 Codex 的内部检查，不要求用户记忆。

## 适用场景

以下任务结束前应执行停止前检查：

- 多文件修改、代码实现、系统规则更新。
- 科研综述、正式报告、PPT、Word、Excel、PDF 等可交付产物。
- 网页资料调研、来源整理、知识库写入。
- 自动化、MCP、外部工具、GitHub 同步或权限相关任务。
- 用户明确批评执行质量、主动性、语气或理解偏差后的修正任务。

## 检查项

Codex 在发送最终回复前应确认：

- 用户真实目标是否被回应，而不只是字面任务被处理。
- 是否已有可用产物、文件修改、知识条目、截图、补丁、报告或明确阻塞。
- 能运行的校验是否已经运行；无法校验时是否说明原因。
- 是否保留了关键来源、文件路径、命令和剩余风险。
- 是否涉及删除、覆盖、外部写入、账号操作或敏感信息；若涉及，是否已确认。
- 是否需要更新用户画像、知识库、复盘记录、工具登记、模板或技能。
- 是否已完成 Git 状态检查、提交和可行的远程同步。
- 是否把下一步负担转移给用户；若 Codex 可继续做，应继续做。

## 未通过时的处理

若检查未通过，Codex 不应直接结束。应继续补齐产物、运行校验、记录阻塞或修正规则。

若外部条件阻塞，例如网络、权限、凭据、账号授权或用户决策，应说明已完成内容、阻塞原因、剩余工作和解除条件。

## 与自动化的关系

稳定、可重复的停止前检查可以转化为脚本、自动化或技能。尚不稳定的流程先保留为人工可读规则，避免过早自动化。

自动化候选应按 `docs/assistant/automation-policy.md` 审查。长任务结束前还应按 `docs/assistant/long-task-quality.md` 检查状态保存、恢复入口和验收维度。
