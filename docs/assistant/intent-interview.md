# 自适应访谈规则

该规则用于让 Codex 在复杂任务前理解真实需求，而不是只按字面指令执行。

## 任务分级

简单任务：输入清楚、输出清楚、风险低。直接执行，不进行额外访谈。

中等任务：目标基本清楚，但存在会影响结果的一到三个关键选择。Codex 应先说明自己对目标的理解，再提出必要问题。若上下文足够，应直接执行。

复杂任务：范围广、文件多、涉及科研判断、正式文档、设计取舍、外部资料、跨工具执行、长期影响或高风险操作。Codex 应进行系统化访谈。

## 访谈内容

复杂任务至少确认：

- 目标：真正要解决什么问题。
- 成功标准：怎样判断结果符合预期。
- 输入：文件、目录、链接、数据和已有背景。
- 输出：文件类型、位置、格式、深度和语言。
- 范围：包含什么，不包含什么。
- 约束：时间、工具、兼容性、隐私和风格。
- 风险：破坏性操作、外部写入、不确定信息和验证限制。
- 偏好：受众、语气、详略、示例、图表、可编辑性。
- 验收：测试、渲染、截图、来源、人工检查或审查标准。

## 提问约束

先探索可用上下文，再提问。不要问可以从文件、项目结构、历史记录或公开资料中得到答案的问题。

问题应少而关键。能安全假设的事项，直接说明假设并继续执行。高影响不确定性必须确认。

## 访谈结束

访谈结束后，Codex 应用一小段话复述最终理解，然后继续执行任务。不得把访谈或计划作为最终交付。
