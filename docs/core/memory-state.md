# 记忆与状态协议

本协议规定哪些信息应保存到本地，以及长任务如何恢复。目标是减少用户重复说明，同时避免把未经确认的推断写成事实。

## 记忆分层

- 当前任务状态：写入 `docs/tasks/active.md`，保存目标、已完成内容、来源、命令、产物、未验证判断、阻塞和恢复入口。
- 用户画像：写入 `docs/profile/user-model.md`，只记录稳定偏好、长期工作场景和明确反向偏好。
- 偏好地图：写入 `docs/profile/preference-map.md`，记录已知、待采集和保守默认。
- 任务知识：写入 `docs/knowledge/`，保存科研、项目、网页资料和可复用结论。
- 系统复盘：写入 `docs/knowledge/system-improvement/harness-log.md`，记录误解、验证不足、执行不足、工具失败和修正动作。
- 工具与能力：写入 `docs/references/assistant/tool-registry.md`、`docs/capabilities/` 和相关脚本。

## 写入标准

信息满足以下任一条件时，应考虑写入本地：

- 能减少用户未来重复说明。
- 会影响科研、办公、编码、网页或知识管理质量。
- 是用户明确表达的偏好、反感行为、质量标准或长期目标。
- 是任务中被验证过的来源、结论、路径、命令或模板。
- 是一次失败或低效的原因及修正方式。

不得写入密钥、令牌、账号密码、未授权个人资料或无法说明来源的敏感信息。

## 长任务状态

超过一次工作回合、超过 30 分钟、涉及多来源或多文件的任务，必须维护结构化状态。状态至少包含：

- 当前目标。
- 已读来源。
- 已执行命令。
- 产物。
- 未验证判断。
- 阻塞。
- 下一步。
- 恢复入口。

中断恢复时，先读取 `AGENTS.md`、核心协议、`docs/tasks/active.md`、相关知识条目和 Git 状态，再继续。
