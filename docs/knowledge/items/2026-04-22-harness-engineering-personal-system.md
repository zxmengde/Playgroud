# 摘要

Harness Engineering 的核心启发是：当智能体反复出错或低效时，改进对象不应只限于提示词，而应扩展到运行环境、工具、检查、数据结构、权限和反馈记录。

# 关键信息

个人智能工作系统应把用户偏好、任务流程、工具登记、知识库、权限规则、自动化和复盘记录组织为可持续维护的本地系统。

Codex Skills 适合沉淀专项流程，Memories 适合作为辅助召回，Automations 适合持续跟进任务，MCP 适合连接外部文档和服务。稳定规则仍应写入本地文件，避免只依赖临时对话。

# 事实、推断与不确定性

事实：Codex 支持技能、记忆、自动化和 MCP 等扩展点。

推断：在 Windows 科研和办公场景中，本地文件知识库加专门技能组比单一全局规则更可靠。

不确定性：外部账号和长期后台能力取决于后续授权、凭据管理和安全边界。
